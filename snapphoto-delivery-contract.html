1. Sözleşmenin amacı

Taraflar (PO, UI/UX, ASO, GPO, Web Dev), SnapPhoto v1.0.0’ın mağaza yayını ve kullanıcı deneyimi için gerekli teslimatları netleştirir. Bu belge bir hukuk danışmanlığı yerine geçmez; ürün kapsamı, kabul kriterleri ve sorumluluk matrisi olarak kullanılır.

2. Ürün tanımı (doğrulanmış)

SnapPhoto, kullanıcının cihazında çevrimdışı çalışan bir fotoğraf araç setidir. Hesap oluşturma, bulut yükleme veya uygulama içi satın alma akışı kod tabanında yoktur. Görüntü işleme cihaz üzerinde yapılır; sonuçlar galeriye veya uygulama depolama alanına kaydedilebilir.

2.2 Operasyonel limitler ve formatlar

  • Toplu işlem üst sınırı: 100 fotoğraf (AppConstants.BATCH_LIMIT)
  • Desteklenen çıktı formatları: JPEG, PNG, WEBP, HEIC, PDF
  • Galeri klasör adı: Pictures/SnapPhoto (AppConstants.OUTPUT_FOLDER)
  • Sıkıştırma presetleri: Maximum Saving, Balanced, High Quality, Custom; hızlı presetler: Instagram, Email, Web
  • Kırpma oranları: Free, 1:1, 4:3, 16:9, 3:4, 4:3
  • Filigran: Corner / Diagonal; konumlar: alt-üst köşeler + merkez
  • Geçmiş filtreleri: All, Today, This Week, This Month
  • Geçmiş otomatik silme: Never, 30 gün, 90 gün

2.3 Navigasyon ve ekranlar

  • Alt sekmeler: Home, History, Menu
  • Onboarding: 2 sayfa (ilk açılışta; tamamlanma durumu DataStore’da saklanır)
  • Compare (önce/sonra) ekranı; ayarlardan kapatılabilir
  • Menü: Rate the app, Send feedback, Share app, Settings, Privacy, Storage & history, Accessibility, Help
  • Değerlendirme istemi: başarılı işlem sayıları 3, 8, 15 olduğunda; reddedilirse 30 gün bekleme

2.4 Dil desteği (6 dil + sistem)

English, Türkçe, Español, 中文 (zh-CN), हिन्दी, العربية (RTL). Android localeConfig ile tanımlıdır.

2.5 İzinler (AndroidManifest)

  • READ_MEDIA_IMAGES — fotoğraf seçimi (Android 13+)
  • READ_EXTERNAL_STORAGE — maxSdkVersion 32
  • WRITE_EXTERNAL_STORAGE — maxSdkVersion 28

2.6 Analitik (koşullu)

app/google-services.json dosyası mevcutsa Firebase Analytics ve Crashlytics etkinleşir (FIREBASE_ENABLED build flag). Dosya yoksa analitik no-op implementasyonu kullanılır. Olaylar: process_success, process_failed (mod ve sayım metadata ile).

3. Rol bazlı teslimat sözleşmesi

3.1 Product Owner (PO)

Yükümlülük: v1.0.0 kapsamını aşağıdaki maddelerle sınırlı tutmak ve kabul testlerini onaylamak.

  • 10 editör modunun tamamının release build’de erişilebilir olması
  • Batch limit 100’ün tüm modlarda tutarlı uygulanması
  • Ayarların (dil, tema, varsayılan kalite/format, galeriye kaydet, compare, EXIF strip, geçmiş saklama, haptik, metin boyutu) çalışması
  • Gizlilik mesajının uygulama içi metinle uyumu: işleme cihazda; fotoğraflar sunucuya gönderilmez
  • Play Store’da satılacak sürümün 1.0.0 / versionCode 1 olması

PO kabul kriteri: Internal test track’te API 26–35 cihazlarda smoke test checklist’inin tamamı geçer.

3.2 UI/UX Designer

Yükümlülük: Material 3 tabanlı mevcut arayüzün tutarlılığını ve erişilebilirliğini garanti etmek.

  • Tema: System / Light / Dark
  • İçerik ölçeği: Small (0,92×), Normal (1×), Large (1,12×)
  • RTL: Arapça layout desteği (supportsRtl="true")
  • Ana ekran: 2 sütun feature grid; Watermark, Background blur, Batch rename, HEIC to JPG kartlarında “New” rozeti
  • Compare ekranı: sürükle-karşılaştır slider; istatistik rozetleri
  • Splash + onboarding akışı
  • Launcher ikonu: adaptive icon (beyaz arka plan + PNG foreground)

UX kabul kriteri: Kritik akış (seç → işle → sonuç / compare / galeri) 3 dokunuştan fazla gerektirmeden tamamlanır.

3.2 ASO Specialist (App Store Optimization)

Yükümlülük: Mağaza listelemesini uygulamanın gerçek özellikleriyle eşleştirmek; abartılı veya kodda olmayan iddia kullanmamak.

  • Uygulama adı: SnapPhoto
  • Önerilen anahtar kavramlar (ürün gerçeğine dayalı): offline photo editor, compress, resize, crop, convert, batch, metadata remover, PDF, watermark
  • 6 dilde yerelleştirilmiş mağaza metni (uygulama içi 6 dil ile hizalı)
  • Short description: çevrimdışı, hesap gerektirmez, toplu işlem vurgusu
  • Feature graphic: 1024×500 px; app icon: 512×512 px (Play Console gereksinimleri)
  • Phone screenshots: gerçek uygulama ekran görüntüleri (mockup veya sahte UI yasak — PO onayı)

ASO kabul kriteri: Listing metnindeki her madde §2’deki mod listesinde karşılığı vardır.

3.3 GPO — Google Play Operations

Yükümlülük: Play Console yapılandırmasını teknik gerçeklere göre tamamlamak.

  • Signed AAB yükleme (./gradlew bundleRelease çıktısı)
  • Package name: com.snapphoto.photo.compress.resize.crop.editor
  • Data safety: fotoğraflar cihazda işlenir; hesap yok; uygulama içi satın alma yok
  • Firebase etkinse Analytics/Crashlytics Data safety’de beyan edilir
  • Content rating anketi: fotoğraf düzenleme aracı profiline uygun tamamlanır
  • Gizlilik politikası URL’si: sites.google.com/view/snapphoto-privacy (uygulama içi link ile aynı)
  • Internal testing → production yayın sırası

GPO kabul kriteri: Play Console’da “Ready to publish” engeli kalmaz (politika, imza, metadata).

3.4 Web Developer

Yükümlülük: Dış yüzey web varlıklarını uygulama ve mağaza gereksinimleriyle senkron tutmak.

  • Gizlilik politikası sayfası (PRIVACY_POLICY_URL) erişilebilir ve güncel
  • Politika metni: cihaz-içi işleme, izinler, isteğe bağlı Firebase, iletişim
  • Destek e-postası (uygulama içi feedback): lagomlabs@gmail.com
  • Play Store linki: Play Store listing URL
  • İsteğe bağlı: GitHub Pages’te bu teslimat sözleşmesinin barındırılması

Web kabul kriteri: Gizlilik URL’si uygulamadan açıldığında 404 vermez; HTTPS geçerlidir.
